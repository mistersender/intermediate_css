Nest Selectors

Stylus: 

.parent-class{
  color: blue;

  .child-class {
    color: green;
  }
}
CSS: 

.parent-class{
  color: blue;
}
.parent-class .child-class {
  color: green;
}

Use Variables

Stylus: 

site.border := 1px solid #ccc
.table{
  border: site.border;

  tr{
    border: site.border;
  }
}
CSS: 

.table{
  border: 1px solid #cccccc;
}
.table tr{
  border: 1px solid #cccccc;
}

Not Strict about syntax

Stylus: 

.test {
  color: green
}

.test
  color green

.test
  color: green;
/* etc.. */
CSS: 

.test {
  color: green;
}
.test {
  color: green;
}
.test {
  color: green;
}

Mixins

Stylus: 

.test
  +simplified_bem()
    color: green
CSS: 

.test,
[class^='test--'],
[class*=' test--'] {
  color: green;
}

Functions

Stylus:

Looping & Interpolation

Folder Structure

Organized site-level css

Solves the problem of difficulty locating styles

Base Structure

  • helpers/
  • base/
  • components/
  • utilities/
  • scoped/

helpers/

Where preprocessor code lives.

3 Primary sub-folders:

  • variables/
  • functions/
  • mixins/

Example Helpers File Structure


helpers/
	all.scss
	variables.scss // site-level variables
	mixins/
		all.scss
		some-custom-mixin.scss
	functions/
		all.scss
		color.scss

base/

Where element-level css resides.

Like reset.css

Example Base File Structure


base/
	all.scss // includes all files in directory
	reset.scss
	general.scss

components/

Where each component lives.

ie: .table or .card

Example Component File Structure


components/
	all.scss // includes all files in directory
	table.scss
	card.scss
	heading.scss
	button.scss

utilities/

Where util-level css lives.

ie: pull-left or clearfix

Example Utilities File Structure


utilities/
	all.scss // includes all files in directory
	reset.scss
	pull.scss
	clearfix.scss

scoped/

Where section-specific site-level css lives.

ie: header or sidebar

Example Scoped File Structure


scoped/
	all.scss // includes all files in directory
	header.scss
	footer.scss
	sidebar.scss
Now the main stlyesheet looks like this: 

@import "base/all",
        "helpers/all",
        "components/all",
        "utilities/all",
        "scoped/all";

BEM

Block | Element | Modifier

Solves the problems of Specificity, Context, & Consistency

Block

The name of the component block.

.card

__Element

An element inside of our block

Dilenated by: __

card__title
card__content

--Modifier

a change from the default to a block or element

Dilenated by: --

card--large
card__title--danger

Example

Stylus: 

.card
  border: 1px solid #ccc
  &__title
    display: block;
    background: green;

    &--light
      background: #ccc;
  &__content
    padding: 10px;

  &--danger
    background: red;
HTML: 



  
A Card

  
Some content

Rendered CSS

(IT CSS :thumbsup:)


.card {
  border: 1px solid #ccc;
}
.card__title {
  display: block;
  background: #008000;
}
.card__title--light {
  background: #ccc;
}
.card__content {
  padding: 10px;
}
.card--danger {
  background: #f00;
}

What are BEM's Benefits?

  • Avoids Nesting (single-level specificity)
  • Clear picture of what CSS & HTML are doing
  • Clearly deliniates components
  • Scales marvelously

Namespacing

Giving styles programmatic context.

Solves the problems of Context

Scenario:

This style is being naughty:

.table-row__header

What could happen if I change this style?

c- Component

A Concrete, standalone, implementation specific piece of UI.

ie: c-card or c-button

o- Object

Abstract box-model only piece of UI. (Similar to component, but more reusable)

ie: o-wrapper or o-actions

u- Utility

A class with a single, specific, trustworthy role.

ie: u-pull-left or u-hide

_- Hack

When the worst happens.

ie: _-ie9-just-cant-focus or _-die-ie-die

Namespaces can also delineate modifications

(SMACSS Guideline)

is- has- Modifier

A temporary, optional or short-lived style

ie: is-awesome or has-mullet

t- Theme

Anything within could have a different theme applied.

ie: t-panda-express or t-zaxbys

s- Scoped

Specifically scoped to a single section or page.

ie: s-coupon-table or s-purchase-requirements
